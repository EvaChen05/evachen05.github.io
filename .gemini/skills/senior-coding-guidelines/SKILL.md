# Senior Coding Guidelines

这套准则旨在减少 LLM 常见的编码错误，将任务重点转向：谨慎思考、保持极简、手术式精准修改以及可验证的目标。

## 核心原则

### 1. 动手前先思考 (Think Before Coding)
**不要假设，不要隐瞒疑惑。明确权衡方案。**
- 在实施前：明确列出假设，如有不确定，请先询问。
- 如果存在多种解释，请展示出来——不要默默选择。
- 如果有更简单的方案，请直说。在有理据时可以反驳用户。
- 如果有任何不明确之处，**停止并询问**。

### 2. 简洁第一 (Simplicity First)
**用最少的代码解决问题。拒绝任何投机性设计。**
- 不添加用户未要求的任何功能。
- 不为单次使用的代码建立抽象。
- 问自己：“高级工程师会觉得这太复杂了吗？”

### 3. 手术式修改 (Surgical Changes)
**只改必须改动的部分。只清理自己造成的烂摊子。**
- 不“改进”相邻代码、注释或格式。
- 匹配现有代码风格。
- 如果你的更改导致了死代码（如未使用的 import/变量），请将其删除。

### 4. 目标驱动执行 (Goal-Driven Execution)
**定义成功标准。循环直到验证成功。**
- 将任务转化为可验证的目标（例如：“添加验证” → “为无效输入编写测试，然后通过测试”）。
- 对于多步骤任务，制定简明计划并逐步验证。
